#### Настройка конфигурации bind
Устанавливаем необходимые пакеты:
```yml
apt-get update
```
```yml
apt-get install -y bind bind-utils
```
ИНСТРУКЦИЯ КАК РЕШИТЬ ПРОБЛЕМУ СКАЧИВАНИЯ ЕСЛИ ОНА ЕСТЬ.

<br/>
Если при вводе apt-get update выходит что то похожее, выполняем следуюющие действия.

Заходим в файл resolv.conf
```yml
mcedit /etc/resolv.conf
```
Добавляем туда это строку.
```yml
nameserver 8.8.8.8
```
В итоге в файле resolv.conf должна получится такая картина.
```yml
# Generated by resolvconf
# Do not edit manually, use
# /etc/net/ifaces/<interface>/resolv.conf instead.
nameserver 8.8.8.8
```
ВАЖНО
После того как все сделали, скачали все что надо, делаем все в обратном порядке. Заходим в файл resolv.conf запись которую ввели, сохраняем файл и выходим. Можете продолжать делать задание 10.

Изменяем содержание перечисленных строк в **`/etc/bind/options.conf`** к следующему виду:
```yml
mcedit /etc/bind/options.conf
```
```yml
listen-on { 127.0.0.1; 192.168.100.62; };

forwarders { 77.88.8.8; };

allow-query { 192.168.100.0/26; 192.168.200.0/28; 192.168.0.0/27; };

```
> **`listen-on`** - сетевые интерфейсы, которые будет прослушивать служба
>
> **`forwarders`** - DNS-сервер, на который будут перенаправляться запросы клиентов
>
> **`allow-query`** - IP-адреса и подсети от которых будут обрабатываться запросы

<br/>

Конфигурируем ключи **rndc**:
```yml
rndc-confgen > /etc/rndckey
```
> Делаем вывод в файл, чтобы скопировать оттуда
```yml
cat /etc/rndckey
```
<br/>

Приводим файл **`/etc/bind/rndc.key`** к следующему виду:
```yml
mcedit /etc/bind/rndc.key
```
ПРИМЕР
```yml
//key "rndc-key" {
//  secret "@RNDC_KEY@";
//};

key "rndc-key" {
  algorithm hmac-sha256;
  secret "ЗДЕСЬ КЛЮЧ КОТОРЫЙ СКОПИРОВАЛИ ВЫШЕ";
};
```
Это пример как долно выглядеть
```yml
//key "rndc-key" {
//  secret "@RNDC_KEY@";
//};

key "rndc-key" {
  algorithm hmac-sha256;
  secret "VTmhjyXFDo0QpaBl3UQWx1e0g9HElS2MiFDtNQzDylo=";
};
```
> Первые строки закомментировали
>
> Вставили ключ **rndc**

<br/>

Проверяем на ошибки:
```yml
named-checkconf
```

<br/>

Запускаем и добавляем в автозагрузку **`bind`**:
```yml
systemctl enable --now bind
```

<br/>

Изменяем **`resolv.conf`** интерфейса(Заходим в папку интерфеса и там исправляем resolv.conf, в нашем случаем это интерфейс ens18):
```yml
mcedit etc/net/ifaces/ens18/resolv.conf
```
```yml
search au-team.irpo
nameserver 127.0.0.1
nameserver 192.168.100.62
nameserver 77.88.8.8
search yandex.ru
```

#### Создание и настройка прямой зоны

Прописываем ее в **`/etc/bind/local.conf`**:
```yml
mcedit /etc/bind/local.conf
```
```yml
zone "au-team.irpo" {
  type master;
  file "au-team.irpo.db";
};
```

<br/>

Копируем шаблон прямой зоны:
```yml
cp /etc/bind/zone/localdomain /etc/bind/zone/au-team.irpo.db
```

<br/>

Задаем пользователя и права на файл:
```yml
chown named. /etc/bind/zone/au-team.irpo.db
```
```yml
chmod 600 /etc/bind/zone/au-team.irpo.db
```

<br/>

Приводим его к следующему виду:
```yml
mcedit /etc/bind/zone/au-team.irpo.db
```
```yml
$TTL    1D
@       IN      SOA     au-team.irpo. root.au-team.irpo. (
                                2024102200      ; serial
                                12H             ; refresh
                                1H              ; retry
                                1W              ; expire
                                1H              ; ncache
                        )
        IN      NS      au-team.irpo.
        IN      A       192.168.100.62
hq-rtr  IN      A       192.168.100.1
br-rtr  IN      A       192.168.0.1
hq-srv  IN      A       192.168.100.62
hq-cli  IN      A       192.168.200.14
br-srv  IN      A       192.168.0.30
moodle  IN      CNAME   hq-rtr
wiki    IN      CNAME   hq-rtr
```

<br/>

Проверяем на ошибки:
```yml
named-checkconf -z
```

#### Создание и настройка обратных зон

```yml
mcedit /etc/bind/local.conf
```

```yml
  type master;
  file "100.168.192.in-addr.arpa";

  type master;
  file "200.168.192.in-addr.arpa";
};

  type master;
  file "0.168.192.in-addr.arpa";
```
